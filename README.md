# Sprint Coach

An AI-powered weekly training assistant for sprint athletes (60m / 100m / 200m / 400m).

Built as a learning project on a clean, modern stack:

- **Next.js 16** (App Router, React Server Components, Server Actions) + TypeScript (strict)
- **Tailwind CSS v4** + hand-rolled shadcn-style UI primitives + Radix + lucide-react
- **Local JSON file store** at `.data/store.json` — zero-setup, swap to Postgres later by replacing `db/fileStore.ts`
- **Anthropic Claude** via a configurable gateway URL (`@anthropic-ai/sdk` + `baseURL`) — strict structured output via a forced `tool_use` call, then `zod` validated
- **recharts** for progress visuals
- Single-user mode (no auth) — flip a single function to add auth later

## What it does (MVP)

1. **Onboarding** — 6-step wizard captures profile, primary event, goals, weekly availability, upcoming meets, and active injuries.
2. **AI-generated weekly plan** — the planner builds a 7-day, periodized week using the athlete's context plus the last 7 days of readiness check-ins and the last 14 days of workout logs.
3. **Daily workout detail** — warmup / main / cooldown blocks, coaching notes, and a "why this session" explanation generated by the model.
4. **One-minute logging** — RPE slider, completed checkbox, notes.
5. **Daily readiness** — sleep, soreness, fatigue, mood. A weighted score drives the dashboard and feeds back into the next regeneration.
6. **Progress** — weekly training load, 8-week consistency heatmap, PR list, streak counter.

## What's deliberately not built

- Auth, accounts, payments
- Mobile apps
- Other event categories (data model supports them, UI doesn't)
- AI chat coach
- Wearable integrations
- The "Learn" section is a stub

Each has a clear seam in the code where it can be added later.

## Getting started

### 1. Install

```bash
npm install
```

### 2. Configure environment

Copy `.env.example` to `.env.local` and fill in your Anthropic credentials:

```
ANTHROPIC_API_KEY=...
ANTHROPIC_BASE_URL=https://eng-ai-model-gateway.sfproxy.devx-preprod.aws-esvc1-useast2.aws.sfdc.cl
ANTHROPIC_MODEL=claude-opus-4-7
```

`ANTHROPIC_BASE_URL` points at the internal model gateway; remove it (or set it to `https://api.anthropic.com`) if you want to call Anthropic directly. The `@anthropic-ai/sdk` automatically sends `x-api-key` and `anthropic-version` headers.

That's it — no database to provision. All data lives in `.data/store.json` (gitignored).

### 3. Run

```bash
npm run dev
```

Open <http://localhost:3000> — you'll be redirected to onboarding on first visit.

### 4. Reset

To wipe your local data and start over:

```bash
npm run db:reset
```

## Project layout

- `app/` — routes (RSC pages + co-located client components and server actions)
  - `app/onboarding/` — 6-step wizard + `completeOnboarding` server action
  - `app/home/` — daily dashboard
  - `app/plan/` — week grid and per-workout detail
  - `app/log/[workoutId]/` — quick log form
  - `app/recovery/` — daily readiness check-in
  - `app/progress/` — charts + PRs
  - `app/profile/` — profile summary
  - `app/learn/` — stub
  - `app/actions/` — shared server actions (`plan.ts`, `log.ts`)
- `components/ui/` — shadcn-style primitives (`Button`, `Card`, `Slider`, etc.)
- `components/` — feature components (`WorkoutCard`, `WeekGrid`, `ReadinessForm`, charts, nav)
- `db/`
  - `schema.ts` — plain TypeScript types for every entity (single source of truth)
  - `fileStore.ts` — JSON file-backed store with `readStore()` / `updateStore()` and a write mutex
- `lib/`
  - `ai/planGenerator.ts` — builds the prompt, calls Claude via a forced `tool_use` (the structured-output equivalent), validates with zod, retries once on failure
  - `ai/schemas.ts` — zod source of truth for the weekly plan shape
  - `ai/explainWorkout.ts` — lazy "why" explanations (plain Messages call)
  - `sprintDomain.ts` — events list, season-phase logic, readiness scoring
  - `queries.ts` — typed read helpers used by RSC pages
  - `athlete.ts` — `getCurrentAthleteId()` — swap this to add auth
  - `anthropic.ts` — thin Anthropic SDK wrapper (honors `ANTHROPIC_BASE_URL`)

## Swapping the JSON store for a real database later

Two seams to replace:

1. `db/fileStore.ts` — implement `readStore()` and `updateStore(fn)` against your database of choice.
2. `lib/queries.ts` — the query functions take the same shape today; replace the array `.filter()` / `.find()` calls with real queries.

Everything else (server actions, RSC pages, components) imports types from `db/schema.ts` and doesn't know the storage exists.

## Adding auth later

`lib/athlete.ts` is the single seam — replace `getCurrentAthleteId()` with whatever your auth layer provides (Clerk, Auth.js, etc.).

## Adding event categories later

The schema uses `event_code` as a string and `workout_type` is generic enough for any event. Add new entries to `ALL_EVENT_CATEGORIES` in `lib/sprintDomain.ts` (flip `supported: true`), and broaden the system prompt in `lib/ai/planGenerator.ts` with the new training principles.
